# Веб-приложение для управления мероприятиями внутри вуза

Учебный Django-проект для курсовой работы по теме: «Разработка веб-приложения для организации и управления мероприятиями внутри вуза».

## Стек

- Python, Django
- HTML, CSS, JavaScript, Bootstrap
- MySQL как основная база данных
- PyMySQL, openpyxl, Pillow
- Docker и docker-compose

## Структура

- `config/` — настройки Django, URL проекта, подключение БД через переменные окружения.
- `accounts/` — пользовательская модель, роли, организации, регистрация, вход, профиль.
- `events/` — мероприятия, категории, теги, места, регистрации, файлы, уведомления, комментарии.
- `dashboard/` — административные страницы пользователей, справочников и статистики.
- `templates/` — серверные Django-шаблоны.
- `static/` — CSS и статические изображения интерфейса.
- `events/management/commands/load_demo_data.py` — загрузка демонстрационных данных.

## Запуск без Docker

1. Создать и активировать виртуальное окружение:

```bash
python -m venv .venv
.venv\Scripts\activate
```

2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Создать `.env` на основе `.env.example`.

Для основного режима укажите данные MySQL:

```env
DJANGO_USE_SQLITE=0
MYSQL_DATABASE=university_events
MYSQL_USER=events_user
MYSQL_PASSWORD=events_password
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
```

Для быстрого резервного запуска без MySQL:

```env
DJANGO_USE_SQLITE=1
```

4. Применить миграции и загрузить демоданные:

```bash
python manage.py migrate
python manage.py load_demo_data
```

5. Запустить сервер:

```bash
python manage.py runserver
```

Открыть: http://127.0.0.1:8000/

## Запуск через Docker

```bash
copy .env.example .env
docker compose up --build
```

Контейнер `web` применит миграции, загрузит демоданные и запустит сервер на http://127.0.0.1:8000/.

## Реализованные функции по задачам 3.1-3.5

### 3.1. Базовая структура и шаблоны

Создан Django-проект с приложениями `accounts`, `events`, `dashboard`, `core`. Реализованы общий layout, navbar, footer, Bootstrap-оформление, лента мероприятий, карточка мероприятия, формы входа/регистрации, личный кабинет, страницы создания/редактирования, участники, уведомления и административные страницы.

### 3.2. Аутентификация и роли

Используется Django auth с пользовательской моделью по email. Есть регистрация, вход, выход, защита страниц, роли `student`, `organizer`, `admin`, проверки доступа во views и в шаблонах.

### 3.3. CRUD

Реализовано управление пользователями для администратора, создание/просмотр/редактирование/удаление мероприятий, регистрация и отмена регистрации, список моих регистраций и список участников мероприятия.

### 3.4. Управление мероприятиями и участниками

В карточке мероприятия есть категория, теги, место, организация, лимит участников, изображение, файлы, комментарии, кнопки регистрации/отмены. Повторная регистрация запрещена, лимит участников учитывается.

### 3.5. Файлы, выгрузки и статистика

Организатор и администратор могут загружать файлы мероприятия, выгружать участников в CSV и Excel. Реализованы уведомления при изменении важных параметров события только для зарегистрированных участников. В панели администратора есть статистика по пользователям, мероприятиям, регистрациям, категориям, популярным событиям и местам проведения.
